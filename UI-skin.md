# MMD 中国风 UI 皮肤设计

## 文档定位

- 状态：Draft
- 更新日期：2026-07-16
- 作用：作为 `DESIGN.md` 的皮肤专题补充，定义 MMD 的中国风视觉方向、设计令牌、组件映射和验收标准。
- 产品表面：主窗口、工作区侧栏、编辑器、实时预览、状态栏、模态框、弹出窗口、Mermaid 容器和 Excalidraw 外层界面。
- 证据来源：`DESIGN.md`、`README.md`、`ROADMAP.md`、`src/styles/base.css`、`src/styles/app-shell.css`、`src/styles/responsive.css`、`src/styles/markdown-preview.css`、`public/styles/typora-theme/typora-jinxiu.css`。

## 设计目标

1. 中国风来自色彩秩序、字体层级、留白和细节比例，而不是大面积纹理、宫廷插画或仿古纸背景。
2. 保持 MMD 安静、精确、本地优先的桌面工具气质，不把编辑器做成展示型网页。
3. 每套皮肤同时覆盖应用外壳、CodeMirror 编辑区和 Markdown 预览，避免只换标题颜色的“半主题”。
4. 保持文件类型、成功、警告、错误等语义颜色稳定，不能为了风格牺牲状态辨识度。
5. 所有皮肤共享布局、控件尺寸、图标和交互，不因换肤产生位置跳动或重新学习成本。

## 非目标

- 不改变文件树、Files/Outline Tab、编辑器、预览和弹窗的信息架构。
- 不引入新的组件库、CSS-in-JS 或主题框架。
- 不使用渐变、木纹、宣纸纹、云纹大底图、山水插画背景或动态粒子。
- 不使用难读的书法字体承担正文、按钮、文件名或状态信息。
- 不让 PDF、DOCX、图片、音频和视频的真实内容被皮肤滤镜改变。
- 不改变 Excalidraw 场景本身的透明背景和用户选择的元素颜色。

## 共享设计原则

### 1. 器物感，不做布景感

皮肤应像青瓷、刻本、青花或朱砂印泥的配色秩序，而不是把整个工作区装饰成古建筑场景。表现手段限制为：纯色、1px 细线、局部 2px 强调线、小面积印章色和克制的阴影。

### 2. 文档优先

- 应用外壳使用中性低饱和表面，颜色重点只落在选中项、焦点、标题和关键状态。
- 编辑器与预览必须拥有最高对比度和最稳定的阅读背景。
- 文件树图标继续按文件类型使用不同颜色，不被皮肤统一染成单色。

### 3. 古意来自排版

- UI 外壳继续使用 `-apple-system`、`SF Pro Text`，保证桌面原生感。
- Markdown 正文沿用 `Noto Serif SC` 与现有锦绣排版。
- 代码沿用 `SF Mono`、Menlo 和等宽字体回退。
- 只允许在预览 H1、空状态标题等低频位置使用更强的宋体气质；不新增网络字体依赖。

### 4. 皮肤之间结构一致

- 工具栏 48px、面板标题 38–40px、树行与按钮 28–32px。
- 圆角仍以 5–8px 为主，弹窗最大 10px。
- Files 与 Outline 保持同级互斥 Tab。
- 所有图标继续使用 Lucide；Vim 按钮保留 Vim 品牌标志。
- Hover、active、focus、selected、disabled 的面积与位置完全一致，只替换令牌。

## 主题令牌模型

### 现有令牌继续保留

`--bg`、`--toolbar`、`--sidebar`、`--panel`、`--panel-strong`、`--panel-muted`、`--border`、`--border-strong`、`--chrome-text`、`--text-main`、`--text-muted`、`--text-faint`、`--text-reading`、`--accent`、`--accent-hover`、`--theme-emphasis`、`--theme-emphasis-foreground`、`--focus-ring`、`--selection-muted`、`--danger`、`--warning`、`--success`。

### 建议新增令牌

| 令牌 | 用途 |
| --- | --- |
| `--editor-bg` | CodeMirror 主背景 |
| `--editor-gutter-bg` | 行号栏背景 |
| `--editor-gutter-text` | 行号文字 |
| `--editor-active-line` | 当前行背景 |
| `--editor-caret` | 光标和 Vim 光标强调 |
| `--preview-bg` | Markdown 预览阅读背景 |
| `--code-bg` | 行内代码和代码块背景 |
| `--code-border` | 代码块边框 |
| `--motif-color` | 细线、标题小标记等非语义装饰 |
| `--mermaid-primary` | Mermaid 主节点色 |
| `--mermaid-secondary` | Mermaid 次节点色 |
| `--mermaid-line` | Mermaid 连接线与文字色 |

### 应用方式

在根节点使用稳定的皮肤标识，不把皮肤类散落到组件内部：

```html
<html data-skin="jinxiu-zhusha" data-appearance="light">
```

皮肤标识建议固定为：

- `jinxiu-zhusha`
- `ruyao-tianqing`
- `qinghua-jilan`
- `songke-zhuying`
- `shanshui-yemo`

## 皮肤一：锦绣·朱砂

### 定位

现有锦绣主题的正统升级版，也是建议默认皮肤。灵感来自朱砂印、白瓷、湖蓝和旧铜色。整体明确、端正，最适合技术写作与课程文档。

### 组件表达

- 选中树行与活动 Tab 使用朱砂色浅底，文字使用深朱砂；只有强选中状态使用实色朱砂配白字。

- 状态栏保持中性，Saved 使用稳定的语义绿色，不改成朱砂。
- Mermaid 使用朱砂主节点、湖蓝次节点、墨色连接线。

### 适用场景

默认写作、课程讲义、产品文档、技术博客、中文长文。

## 组件映射

### 应用工具栏

- 保持紧凑单行，不增加主题名称或装饰文案。
- 品牌标记使用 `--accent`，不得使用渐变。
- Saved、Edited、Working 继续使用语义颜色；皮肤只调整其与背景的对比。

### 工作区侧栏

- Files 与 Outline 是互斥 Tab，共用 `--accent` 和 `--selection-muted`。
- 活动文件的“已打开”圆点与选中背景必须保持两个独立信号。
- 树层级线只使用 `--border` 或 `--motif-color`，避免彩色层级线干扰扫描。

### 编辑器

- CodeMirror 背景、行号栏、当前行、选区和光标全部由主题令牌控制。
- Markdown/HTML 语法高亮采用共享的可访问语义色板，不为每套皮肤重做完全不同的语法颜色。
- Vim 状态与光标仍需清楚；Vim 图标保持品牌绿色，仅调整周围按钮状态。

### Markdown 预览

- `--theme-emphasis` 负责 H1、H2、链接和引用等锦绣主题强调。
- 皮肤可以调整标题、引用、表格、代码块和 Mermaid 色板，但不能改变 Markdown 结构和语义。
- 正文宽度、字号和行距属于阅读偏好，不随皮肤变化。

### 模态框

- 背景、边框、文字和普通图标跟随皮肤。
- 错误、警告、危险操作继续使用固定语义色。
- 遮罩统一为中性半透明黑，不使用皮肤强调色染色。

### Mermaid 与 Excalidraw

- Mermaid 从主题令牌生成固定配置，保证离线渲染与导出一致。
- Excalidraw 只换外层工具界面；用户场景、透明画布和元素颜色不被皮肤覆盖。
- 导出 SVG/PNG 时默认按文档内容主题处理，不把应用外壳颜色写入图像。

## 交互与持久化

- 在 `View` 菜单或“外观”设置中提供单选皮肤列表，当前皮肤带勾选状态。
- 皮肤切换立即生效，不弹成功提示。
- 皮肤选择写入本地设置；主窗口与编辑器/预览弹出窗口保持同步。
- “跟随系统”只控制 light/dark appearance，不自动在五套皮肤之间轮换。
- 如果用户选择亮色皮肤并启用“跟随系统”，夜间使用对应的安全暗色回退；首版可统一回退到“山水·夜墨”。
- 切换皮肤不得重建编辑器状态、移动光标、重置滚动位置或重新打开文件。

## 可访问性要求

- 普通文字对背景对比度至少 4.5:1，大字号至少 3:1。
- 选中行、活动 Tab 和主要按钮不能只靠颜色表达，继续保留 `aria-selected`、`aria-pressed` 和图形/文字状态。
- 焦点环在每套背景上清晰可见，宽度保持 3px。
- 深色皮肤中语义红、黄、绿需单独校准，不直接沿用亮色数值。
- `prefers-reduced-motion` 继续禁用非必要动画；换肤不增加过渡动画。
- 颜色不是文件类型、保存状态或错误状态的唯一辨识手段。

## 响应式要求

- 窄窗口只调整布局，不改变皮肤色彩层级。
- 侧栏收起后，图标按钮必须保持皮肤下的 Hover、active 和 focus 状态。
- 编辑器与预览堆叠时，分隔边框仍需可见，不能因相邻表面同色而消失。
- 皮肤切换控件在窄窗口进入菜单或模态设置，不新增横向滚动工具栏。

## 实施顺序

1. 把 `base.css` 的现有颜色替换为可覆盖的主题令牌，并补齐编辑器、预览和 Mermaid 令牌。
2. 实现 `data-skin` 与 `data-appearance`，先落地“锦绣·朱砂”和“山水·夜墨”。
3. 将 CodeMirror、Markdown 预览、模态框和弹出窗口接入同一主题状态。
4. 增加本地设置持久化和窗口间同步。
5. 依次加入汝窑、青花和宋刻皮肤，逐套完成截图与对比度验收。
6. 更新 `DESIGN.md`：移除上一次视觉刷新中“Markdown 渲染样式不可变”的临时边界，明确皮肤令牌可以改变视觉但不能改变渲染语义。

## 测试与验收

### 自动化

- 每套皮肤的令牌必须完整，缺少令牌时测试失败，不允许静默回退为随机旧值。
- 设置解析拒绝未知皮肤标识，并回退到默认“锦绣·朱砂”。
- 重启后恢复已选皮肤；主窗口和弹出窗口收到同一主题状态。
- 切换皮肤不改变活动文件、dirty 状态、光标位置、编辑器内容和预览滚动目标。
- Mermaid 在每套皮肤下生成非空 SVG，错误图表仍保持可读回退。

### 视觉验收

- 桌面宽窗口：主窗口、Files、Outline、编辑器、预览和状态栏。
- 窄窗口：侧栏、堆叠面板、搜索框、状态栏与弹出窗口。
- 组件状态：default、hover、focus、selected、active、disabled、loading、error、warning、success。
- 内容样本：中文长文、英文、表格、代码、KaTeX、Mermaid、图片、PDF、DOCX 和 Excalidraw。
- 每套皮肤至少提供亮度直方图与正文/按钮/选中行的对比度记录。

## 推荐决策

- 首发默认：锦绣·朱砂。
- 首发暗色：山水·夜墨。
- 第二批：汝窑·天青、青花·霁蓝、宋刻·竹影。
- 首版不提供用户自由配色，避免产生无法测试的组合；后续可开放“强调色”有限选项。
