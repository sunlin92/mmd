# MMD Roadmap

更新日期：2026-07-15

本文档以当前代码为基线，并合并产品功能清单中的目标。它只把有代码和测试证据的能力标记为已完成；体积、内存和跨平台体验等指标必须由正式构建与实测验证。

## 产品目标

MMD 的目标是成为一个本地优先、打开迅速、实时预览、跨平台一致的桌面 Markdown 编辑器。核心体验按以下顺序建设：

1. 写作闭环：打开、编辑、预览和保存可靠顺畅。
2. 内容表达：GFM、代码、数学公式、Mermaid、Excalidraw 手绘图和本地图片完整可用。
3. 输出交付：可离线导出完整样式的 HTML、高清长图及可编辑图表源文件。
4. 桌面集成：双击打开、CLI、文件关联、自动更新和三平台发布。

## 交付路线

### R1 写作效率闭环（P0）

目标：先补齐高频写作流程，使长文导航和编辑状态可靠可控。

- [~] 大纲侧栏：H1-H6 层级树和点击后编辑/预览定位已完成；当前标题高亮与滚动位置同步未完成。
- [ ] 自动保存：可开关、可配置延迟、原子写入；保存中/已保存/失败状态可见。
- [x] 状态栏：实时字数、字符数、行数和当前光标位置。
- [ ] 主题模式：亮色、暗色、跟随系统一键切换并持久化。
- [ ] 基础偏好设置：字号、行高、内容宽度、Tab 宽度和自动换行。

验收标准：

- 大纲对重复标题、中文标题和编辑中的标题更新稳定。
- 自动保存失败必须用用户友好模态框反馈，并保留 dirty 状态。
- 主题切换覆盖应用壳、编辑器、Markdown 预览和弹出窗口。

### R2 富内容与编辑辅助（P0）

目标：覆盖技术文档常见内容，并减少重复 Markdown 输入。

- [~] Mermaid：离线渲染流程图、时序图、甘特图和状态图；错误图表当前回退为源代码块，专用说明占位未完成。
- [x] Excalidraw 文件类型：在工作区识别、创建、打开和保存标准 `.excalidraw` 场景文件。
- [~] Excalidraw 手绘编辑器：支持画布缩放/平移、选择、图形、文本、连接线和撤销重做；自动保存未完成。
- [~] Excalidraw 中文排版：新建文本默认 `fontFamily: 5`；Maple Mono NF CN 可用性检查和缺失字体提示未完成。
- [ ] Excalidraw 插入 Markdown：可插入与源场景关联的 SVG/PNG，源文件更新后可重新渲染，避免手工维护两份内容。
- [ ] Word/PDF 富文本智能粘贴：从 Word、PDF 阅读器复制内容到 Markdown 编辑器时，自动转换为 Markdown，并尽可能保留标题、段落、粗体、斜体、列表、引用、链接、表格、代码和图片结构。
- [ ] 图片粘贴：从剪贴板写入工作区，按 MD5 去重并自动插入 Markdown。
- [ ] 图片目录设置：支持相对工作区和显式授权的绝对路径，切换后不破坏已有引用。
- [ ] 编辑器插入菜单：表格、提示块、目录、链接、图片、代码块和公式。
- [ ] 自动补全：括号、引号、代码围栏、列表延续和表格辅助。
- [ ] Emoji 短代码：例如 `:rocket:` 转换或补全为对应 Emoji。
- [ ] 编辑器右键菜单：常用格式、插入和链接/图片操作。

验收标准：

- Mermaid、KaTeX 与 Excalidraw 均完全离线，不从 CDN 加载资源。
- Excalidraw 场景保存前必须校验元素唯一 ID、非零长度连接线、文本字体和形状/文本双向绑定；无效场景不得覆盖最后一份有效文件。
- 中文图表保存并重启应用后可继续编辑，SVG/PNG 导出结果不得出现缺字、乱码或字体静默替换。
- Word 粘贴优先解析剪贴板 HTML/RTF，清理 Office 私有样式与不安全内容；转换结果不得携带脚本、外部跟踪资源或本机临时文件路径。
- PDF 粘贴在剪贴板仅提供纯文本时，应保留阅读顺序、段落和换行，合并行末断词；无法可靠识别的表格或分栏不得伪造为错误结构。
- 富文本粘贴必须作为一次可撤销编辑事务；转换失败时回退为经过清理的纯文本，并通过模态框说明未保留的内容类型。
- 从 Word/PDF 粘贴的图片沿用图片目录、MD5 去重和路径授权规则，不能写入工作区外的未授权位置。
- 图片写入沿用现有路径授权边界，拒绝目录逃逸和未授权绝对路径。
- 同一图片重复粘贴只保留一份文件；撤销编辑不误删共享图片。

#### Excalidraw 实现基线

实现参考本地 `excalidraw-handdrawn` 技能，但桌面应用不依赖技能目录、Python、`uv` 或外部浏览器：

- 使用固定版本的 `@excalidraw/excalidraw`，构建为随应用发布的本地 ESM 资源；升级版本时同步更新场景兼容性测试。
- 以标准 Excalidraw `version: 2` JSON 为唯一可编辑源，保留 `elements`、`appState` 和 `files`，不发明私有 `label` 等字段。
- 形状与 bound text 保持双向绑定；绑定箭头同时登记在两端形状中；连接线必须包含真实且非重合的 points。
- 新建中文文本默认 `fontFamily: 5`；Maple Mono NF CN 作为首选中文字体，并将字体可用性纳入启动前检查与导出前检查。
- 在应用内复用同一 Excalidraw 导出运行时生成 SVG 和 PNG；PNG 支持 1x/2x/3x，SVG 保留字体族信息。
- 每次导出同时保留 `.excalidraw` 源文件；“分享图表”默认生成 `.excalidraw`、`.svg`、`.png` 三件套。
- 默认透明画布、`roughness: 1`、正文不小于 18 px、标题不小于 28 px，并提供小型一致的浅色调色板。

### R3 导出与分享（P1）

目标：让预览结果可以直接交付，不依赖接收方安装 MMD。

- [ ] 导出单文件 HTML：内联主题、代码高亮、KaTeX、Mermaid 和可内联图片。
- [ ] 导出高清长图 PNG：支持缩放倍率、最大尺寸保护和进度反馈。
- [ ] 导出 Excalidraw 三件套：一次操作生成可编辑 `.excalidraw`、可缩放 `.svg` 和 1x/2x/3x `.png`。
- [ ] 导出主题选择：亮色、暗色或当前主题，确保中英文与 Emoji 字体回退一致。
- [ ] 导出前检查：缺失图片、超大文档和图表错误通过模态框集中反馈。

验收标准：

- 导出的 HTML 在断网环境和新浏览器会话中完整显示。
- HTML 不执行 Markdown 中未受信任的脚本，外部链接保持安全属性。
- 长图对表格、代码、公式、Mermaid、Excalidraw 和跨页长文无截断。
- Excalidraw 的 SVG 与 PNG 必须在同一离线导出会话生成，并在写盘前完成非空尺寸和可解码性检查。

### R4 桌面集成与发布（P1）

目标：兑现“像原生文档应用一样打开和分发”。

- [ ] `.md` / `.markdown` 文件关联与操作系统启动文件事件。
- [ ] CLI：`mmd <file-or-directory>`，复用已运行实例并聚焦目标文档。
- [ ] 静默检查更新，有新版本时使用模态框提示；支持跳过版本。
- [ ] macOS、Windows、Linux CI 构建、冒烟测试和安装包产物。
- [ ] 代码签名、公证、更新签名和发布回滚说明。

验收标准：

- 三个平台均可通过文件双击、CLI 和应用内菜单打开同一文档。
- 已运行实例收到新文件时不重复启动主窗口、不丢当前未保存状态。
- 更新检查失败保持静默，不影响离线编辑和启动速度。

### R5 性能、可靠性与专业化（P2）

目标：用可复现指标验证速度、资源占用和长期稳定性。

- [ ] 启动性能基准：冷启动、暖启动、首屏可编辑和首个预览完成时间。
- [ ] 大文档基准：1 MB、5 MB Markdown，以及大量代码块、公式和图表场景。
- [ ] 大画布基准：100、500、1,000 个 Excalidraw 元素的加载、编辑、保存和 3x PNG 导出。
- [ ] 内存基准：空闲、普通文档、大文档和 PDF/DOCX 预览分别记录。
- [ ] 安装包体积预算：按平台记录应用、安装包和主要依赖占比。
- [ ] 可自定义快捷键，并检测冲突、支持恢复默认。
- [~] 崩溃恢复：已恢复上次工作区和最后一个已提交文件；未保存草稿、窗口布局和恢复选择未完成。

性能目标（待实测，不是当前承诺）：

- 双击到可编辑：常规设备暖启动目标小于 1 秒。
- 普通 Markdown 编辑：输入到预览更新的 P95 目标小于 100 ms。
- 空闲内存：以 release 构建实测后设定平台预算；不预先宣称 `<50 MB`。
- 安装包：以三平台产物实测后设定预算；当前功能集不预先宣称 `~7 MB`。

## 优先级与依赖

| 顺序 | 能力 | 关键依赖 | 原因 |
| --- | --- | --- | --- |
| 1 | 大纲、自动保存、状态栏 | 现有外部变更监听 | 构成完整长文写作闭环 |
| 2 | 主题与偏好设置 | 共享设置存储和弹出窗口同步 | Mermaid、导出需要统一主题来源 |
| 3 | Mermaid、Excalidraw、富文本粘贴与图片工作流 | 渲染隔离、剪贴板清理、资源授权、字体检测和持久化设置 | 完成技术文档迁移与可视化表达能力 |
| 4 | HTML/PNG/SVG 导出 | 稳定的主题、Mermaid、KaTeX、Excalidraw 与图片资源管线 | 避免先做一套不完整导出后返工 |
| 5 | 文件关联、CLI、更新与发布 | single-instance、三平台 CI | 最后形成可靠桌面分发闭环 |

## 明确不做

- 云同步、账号体系和在线协作。
- 在未实测前以安装包体积或内存数字作为营销承诺。
- 为追求“原生”而复制三套独立 UI；保持同一套核心代码与平台适配层。
- 绕过现有 Tauri 路径授权模型访问任意本地文件。

## Roadmap 完成定义

每个条目只有同时满足以下条件才可勾选：

1. 行为已有自动化测试，安全敏感逻辑包含 Rust 测试。
2. `npm run lint`、`npm test`、`npm run typecheck`、`npm run build`、`cargo test --manifest-path src-tauri/Cargo.toml` 全部通过。
3. 用户可见错误使用统一模态反馈，不暴露原始运行时错误。
4. UI 功能在桌面与窄窗口完成截图或真实应用验收。
5. README 使用说明与本文件状态同步更新。
